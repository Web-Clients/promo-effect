# Полный Анализ Платформы Promo-Efect Logistics

Этот документ содержит полное описание архитектуры, функционала и бизнес-процессов платформы.

## 1. Техническая Структура (Tech Stack)

Платформа построена как современное веб-приложение:

*   **Frontend (Клиентская часть):**
    *   **React + TypeScript:** Обеспечивает быструю работу интерфейса без перезагрузки страниц.
    *   **Tailwind CSS:** Используется для дизайна (стилистика).
    *   **Lucide React:** Набор иконок.
*   **Backend (Серверная часть):**
    *   **Node.js + Express:** Сервер обработки запросов API.
    *   **Prisma ORM:** Инструмент для работы с базой данных (безопасные запросы).
    *   **PostgreSQL:** Основная база данных для хранения пользователей, цен, заказов.
*   **Инфраструктура:**
    *   **PM2:** Менеджер процессов (держит сервер включенным).
    *   **Nginx:** Веб-сервер (раздает файлы сайта).

## 2. Роли Пользователей

В системе есть 3 основных уровня доступа:

### 👑 1. ADMIN (Администратор)
Полный контроль над системой.
*   **Управление Агентами:** Создание, удаление, блокировка китайских агентов.
*   **Управление Ценами:**
    *   Настройка базовых цен (Freight) от лица платформы.
    *   Настройка портовых сборов (Constanța/Odessa) и наземного транспорта.
    *   Утверждение/Отклонение цен, присланных агентами.
    *   Редактирование цен агентов напрямую (без одобрения).
*   **Управление Заказами:** Просмотр всех бронирований клиентов, смена статусов.
*   **Финансы:** Установка маржи (Profit Margin), комиссий и курса валют.

### 🏢 2. AGENT (Китайский Агент)
Партнер, предоставляющий цены на перевозку.
*   **Добавление Цен:** Может предлагать свои цены на фрахт (Port-to-Port).
*   **Статусы Цен:** Цены агента требуют одобрения Админом (статус `PENDING` -> `APPROVED`).
*   **Просмотр Заказов:** Видит только те заказы, которые клиенты сделали по *его* ценам.

### 👤 3. CLIENT (Клиент)
Конечный пользователь, который ищет доставку.
*   **Калькулятор:** Ищет стоимость доставки (вводит порт и пункт назначения). Система автоматически подбирает лучшую цену (Базовая или Агентская + Маржа Платформы).
*   **Бронирование:** Создает заявку на перевозку.
*   **Отслеживание:** Видит статус своего груза.

## 3. Основные Модули и Процессы

### A. Ценообразование (Pricing Engine)
Сердце платформы. Итоговая цена для клиента складывается автоматически:
1.  **Морской Фрахт (Freight):** Берется либо базовая цена платформы, либо цена агента.
2.  **+ Портовые Сборы:** Автоматически добавляются настройки порта прибытия (например, Констанца $221.67).
3.  **+ Наземная Доставка:** Авто-расчет до города клиента (на основе регионов/расстояний).
4.  **+ Маржа и Комиссия:** Надбавка платформы (настраивается в Admin -> Settings).
5.  **= Итоговая цена.**

### B. Модуль Агентов (Agents Module)
*   **Создание:** Агентов создает ТОЛЬКО Админ вручную.
*   **WeChat:** Поле информационное, интеграции нет.
*   **Agent Portal:** У агентов есть свой личный кабинет (упрощенный), где они управляют только своими ценами.

### C. Бронирование и Заказы (Bookings)
1.  Клиент делает поиск в калькуляторе.
2.  Выбирает вариант и жмет "Бронировать".
3.  Заказ падает в статус `PENDING`.
4.  Админ (или Агент) подтверждает наличие места -> статус `CONFIRMED`.
5.  Далее статусы меняются по ходу движения: `IN_TRANSIT`, `CUSTOMS`, `DELIVERED`.

### D. Калькулятор (Calculator)
*   Умный алгоритм, который сравнивает все доступные цены в базе.
*   Учитывает тип контейнера (20ft/40ft), вес груза (Weight Ranges) и дату отправки.

### E. Админ-Панель (Admin Dashboard)
*   **Статистика:** Графики выручки, новые клиенты.
*   **Управление:** Все настройки системы вынесены в интерфейс (код менять не нужно).

## 4. База Данных

Система хранит:
*   **Users:** Все пользователи и их хешированные пароли.
*   **Agents:** Профили агентов (связаны с Users).
*   **AgentPrices:** Цены, предложенные агентами.
*   **BasePrices:** Цены самой платформы.
*   **Bookings:** Заказы.
*   **AdminSettings:** Глобальные настройки (налоги, маржа).
